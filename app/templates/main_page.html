Home

Welcome to the central hub, from here you can complete all tasks.

First, we will create either a blank spaCy language object or clone from an existing object.

From there, we will load a corpus of texts to test the existing defaults.

1. Create A New Language Object

Create or clone a new custom spaCy language object for your project

2. Sentences

Add several example sentences that will be used for initial tokenization.

3. Tokenization

During tokenization, a text is split into its constituent parts. The spaCy tokenizes texts using a default algorythm. (docs)

4. Lookups and Matching

Cadet serves automatic suggestions to INCEpTION to create model training data. To facilitate annotation, Cadet can serve suggestions from lists of lemmata, part-of-speech or entities.

5. Create a Corpus to identify Stop Words and Frequent Terms

Cadet uses an initial corpus of texts to identify frequent terms for annotation. The texts are used to test the existing defaults in spaCy for use with your language.

6. Identify and annotate stop words and frequent terms

Using the corpus from the earlier section, add data to frequent terms to generate auto-suggestions.

7. Export to INCEpTION, Serve Auto-suggestions

Configure endpoints to serve suggestions to INCEpTION.

8. Fetch data from INCEpTION, Assess training data

Connect to INCEpTION to gather annotation data, save it as spaCy training data and debug to assess if the data is ready.

9. Export Language Object and Training Data

The final step. Cadet exports files for the language object, its lookups data and training data from INCEpTION. It's everything you'll need to begin training model pipelines.
